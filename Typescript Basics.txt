Typescript Basics


TS is superset of JS. It is transcompiled into JS at build/runtime so that all of the type safety syntax is removed

Compiling .ts files
------------------------
To transpile a .ts file to a .js file we have to use the typescript compiler

tsc index.ts // if there are no errors this creates a new file in the folder called index.js


Native TypeScript in Node
---------------------------------
- as of  Node 22.18.0 typescript can be executed natively

If you are using v22.18.0 or later and your source code contains only erasable TypeScript syntax, you can execute TypeScript code without any flags.

node example.ts


Imports
-----------
- method for brining it variables, functions, and objects from other modules in the codebase

[ Named Exports ]
- most common and clean approach
- identify exports where they are defined rather than at the bottom of the file
- can alias named imports 

import { formatCurrency, formatDate } from './utils.ts'
import { formatCurrency as format, formatDate } from './utils.ts'
import * as data from './utils.ts' // import everything and asscociate with data object

[ Default Exports ]
- least common
- used in React Component libraries
- use when the module has one clear purpose and you are exporting one thing
- when I import I can call the export whatever I  want at import time

export default class Formatter {
	formatCurrency(amount: number): string {
		return formatCurrency(amount)
	}

	formatDate(date: Date): string {
		return formatDate(date)
	}
}

import Formatter from './utils.ts'

Interfaces
-------------
- another way to name an object type
- define a "contract" that our objects will follow. 
- this happens solely at compile time; once the code is compiled the concept of interfaces is gone.

interface IProduct {
    id: number;
    name: string;
    description: string;
    price: number;
}

// create an instance of IProduct
var product:IProduct = {
    id: 1,
    name: "Call of Duty",
    description: "The latest COD game",
    price: 59.99
}


interface PaymentMethod {
	pay(amount: number): string
}

class CreditCard implements PaymentMethod {
	cardNumber: string

	constructor(cardNumber: string) {
		this.cardNumber = cardNumber
	}
   // because the CreditCard class implements (or extends) PaymentMethod we MUST provide a concrete implementation of the pay method
	pay(amount: number): string {
		return `Paid $${amount} with credit card ${this.cardNumber}`
	}
}

[ Type Guarding ]
- validates that something that was given to me (an argument) is of a specific type

# type guard function
function isStringArray(value: TextInput): value is Array<string> {
	return Array.isArray(value) && value.every((item) => typeof item === 'string')
}

# use the type guard function
# since it will be of type array or string, i implement logic that assumes the join and trim methods are available
function normalizeText(input: TextInput) {
	if (isStringArray(input)) {
		return input.join(' ').trim()
	}
	return input.trim()
}

[ Discriminate Unions ]
- this is when you declare a family of types and use a literal property (.e.g kind) to differentiate between them
- all other properties will use primitive types (.e.g. string, number)
- you then declare a higher order type that is a union of the family of types
- I can then use that property as a "checker" to determine which type I am using in a particular situation

type Circle = { kind: 'circle'; radius: number }
type Rectangle = { kind: 'rectangle'; width: number; height: number }
type Shape = Circle | Rectangle


[ Literal Types ]
THIS IS POWERFUL CONTINUE TO WORK ON THIS
- deriving types from variables at runtime using "as const" at the end of an object definition
- this is similar to defining enums but it is more dynamic and fluid

const routes = {
	home: '/',
	login: '/login',
	settings: '/settings',
} as const


// Create a `RouteName` type using `keyof typeof routes`
type RouteName = keyof typeof routes // takes the keys of the object and creates a union out of them

// Create a `RoutePath` type from the values of `routes`
type RoutePath = (typeof routes)[RouteName] // takes the values of each of the keys in the object and makes a union out of them 

// Add `as const` so this stays the literal '/'
const defaultRoute: '/' = routes.home

// Implement `getRoutePath` to return the path for a route name
function getRoutePath(route: RouteName): RoutePath {
	return routes[route]
}

console.log('Value: ', getRoutePath('settings'))

Generic Functions
---------------------
- allows me to compose reusable functions that can be conumed using many different Types
- <T> represents a "Type Variable" that signals to TypeScript that the type that is passed into the function should also be returned 
- this provides us flexibilty with type safey unlike using "any" which provides us flexibility with no guarnteed type safety

function identity<T>(value: T): T {
	return value
}

// Create a generic `last` function
// Takes an array of Item, returns Item | undefined
function last<T>(list: Array<T>): T | undefined {
	return list.at(-1)
}

const createRow = <T,>(label: string, value: T, disabled = false) => ({
  // do stuff
});
