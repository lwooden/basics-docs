Tanstack Basics


What is Tanstack
----------------------
- At its core, TanStack is a collection of tools that help you build modern apps
- Indepenent components such as Tanstacker Router, data fetching and caching, server functions (SSR),
forms, and AI can be combined to build the app that I want
- TanStack Start is a full framework, built on top of Router that performs SSR, streaming, and server functions
- TanStack Router and the other supporting components ARE NOT

// Mental Model
TanStack Router = lightweight SPA application (think Pure React)
TanStack Start = full-stack application

// Type Safety
One more thing that sets TanStack apart: type safety.
My routes, loaders, params, and search params are all fully typed, automatically.
So if my loader returns a list of posts, my component already knows the shape of that data.

// General Request Flow

/posts (Route) -> Data Fetch   -> Cache    -> UI Render
User Nav 						Loader runs		 Store Result			Display

Understanding Loaders
-------------------------------
- createServerFn defines server-only logic. The loader calls it, on the server during SSR, it runs directly; on the client during navigation, it becomes an RPC call to the server.
- the component AWLAYS receives data before rendering

import{createFileRoute}from'@tanstack/react-router'
import{createServerFn}from'@tanstack/react-start'

// Server Function Definition
const getPosts = createServerFn().handler(async () => {

# Runs on the server only — safe to use env vars, DB calls, etc.

const res = await fetch('https://api.example.com/posts')
return res.json()

})


export const Route = createFileRoute('/posts')({
loader: () => getPosts(),  # reference server function

component: PostsPage # define UI component to render
})

// Component Definition
function PostsPage() {
const posts = Route.useLoaderData() # used to access what the loader returned (type safe)

return (
<ul> {posts.map(post => (<li key={post.id}>{post.title}</li>))} </ul>

 )
}
